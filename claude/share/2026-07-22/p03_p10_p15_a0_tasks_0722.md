## P03-A0: Temporal Connectivity Gap（推荐首选）

**假设：** 流式插入使新节点优先连接其他新节点，跨时段边密度显著低于静态构建。

**实验：**
1. SIFT1M 分 4 个 cohort（每 250K 为一段 C0-C3）
2. **流式构建：** 按序插入 C0→C1→C2→C3，每段插完后导出边列表
3. **静态构建（oracle）：** 用全部 1M 随机 shuffle 后一次性构建
4. **度量：** 4×4 跨段边密度矩阵 `D[i][j] = edges(Ci→Cj) / (|Ci|×R)`，对比流式 vs 静态
5. **搜索质量：** 对 10K 查询，按其 GT 1-NN 所在 cohort 分组报 recall

**PASS：** 流式构建的 D[C0][C3] < 0.5 × 静态构建的 D[C0][C3]  
**KILL：** 流式与静态的 D 矩阵差异 <20%

**代码基础：** 可复用 `codex/work/2026-07-22/graph_aging_a0/PipeANN/tests/graph_aging_a0.cpp`（已有分批插入 + 边导出 + recall 测量）  
**数据：** SIFT1M `full_1m.bin`  
**时间：** 2-3h

## P10-A0: PQ Navigation Corridor Divergence

**假设：** PQ 距离误差在搜索初期复合放大，使 beam 进入错误走廊，与精确距离导航的 visited-node 集合显著不同。

**实验：**
1. 在 DiskANN 内存图上运行两种 beam search：
   - A: PQ 距离导航（标准）
   - B: 精确 L2 距离导航（oracle，需加载全向量到内存）
2. 对 10K 查询，记录 A 和 B 各自的 visited-node 集合
3. **度量：** visited-node Jaccard(A, B)；"divergence depth" = A 和 B 路径分离的平均 hop 数
4. **搜索质量：** recall@10 of A vs B

**PASS：** Jaccard < 0.6（>40% 路径分离）且 recall 差 > 3pp  
**KILL：** Jaccard > 0.85 或 recall 差 < 1pp

**实现：** 需在 DiskANN `Index::search` 基础上添加 exact-distance 导航模式。工程量中等。  
**时间：** 3-4h

## P15-A0: Approximate Freshness Threshold

**假设：** 未连接到图的 pending 插入数 N 与 recall 退化之间存在非线性相变。

**实验：**
1. SIFT1M 静态构建 800K 的内存图
2. 将剩余 200K 分批"插入"但 **不连接到图**（只加入数据，不执行 prune/双向边）
3. 在每个 pending 级别（N = 0, 10K, 20K, 50K, 100K, 150K, 200K）运行查询，GT 包含所有已插入向量
4. **度量：** recall@10 vs N 曲线；寻找拐点（二阶导数极值）

**PASS：** 存在明确拐点（某 N 值前后 recall 变化率 >3×）  
**KILL：** recall 线性下降或 monotone 无拐点

**实现：** 需要"unconnected insert" API。现有 `DynamicIndex::insert` 会自动搜索+prune+连边，需绕过或修改。工程量中等。  
**时间：** 2-3h
