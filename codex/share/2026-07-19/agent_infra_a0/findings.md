## 2. T1 raw data and claim audit

### 2.1 Recomputed comparison

The published event-level ratios are arithmetically correct, but compare one corpus-wide event affecting 1,013 objects against one local mutation:

| Layout | Upgrade total | Mean non-upgrade total | Event ratio | Upgrade per object | Mean non-upgrade per object | Per-object ratio |
|---|---:|---:|---:|---:|---:|---:|
| A | 3.93 MiB | 12.42 KiB | 324.19× | 3.98 KiB | 9.66 KiB | 0.412× |
| B | 2.02 MiB | 32.53 KiB | 63.57× | 2.04 KiB | 25.05 KiB | 0.0815× |
| C | 2.18 MiB | 22.54 KiB | 98.91× | 2.20 KiB | 17.88 KiB | 0.123× |

Thus the headline 64–324× “dynamic range” is the registered normalization problem in another form: an `O(N)` corpus rewrite is compared with an `O(1)` operation. Under the revised plan's required `B_per_obj`, upgrade is only 8–41% of the weighted non-upgrade mean. This does not imply upgrade is cheap in total, but it does invalidate the claimed per-object semantic amplification signal.

### 2.2 `B_abs` is not an absolute write counter

The code defines:

```text
B_abs = sum(max(0, file_size_after_FULL_checkpoint - file_size_before))
```

over SQLite main, WAL, and rollback-journal files. Every operation starts after a WAL truncate and ends with `FULL` checkpoint followed by another truncate. Therefore:

* WAL extent is recreated for each operation and counted;
* overwrites to already allocated main-file pages during checkpoint are not counted;
* filesystem writeback and host block submissions are not counted;
* the forced checkpoint/truncate sequence changes the workload being measured.

The claimed 96–99% “WAL dominates B_abs” is consequently mechanical: the numerator counts WAL extent while the main-file term counts only net growth, not main-page overwrite traffic. It is not evidence that 96–99% of application, filesystem, block-device, or NAND writes come from WAL.

### 2.3 Registered closure fields are absent

The 3,300 raw rows contain none of the following:

```text
B_ratio
pre/post canonical state hash
W_app or LD_PRELOAD syscall ledger
W_fs/cgroup writeback
W_blk/block trace
unattributed-byte bucket
```

`raw_application_bytes` is a Python estimate of serialized values passed to logical operations, not observed write syscalls. `DBSTAT` is only used to compare table page-count/payload deltas for Layout B; it does not identify pages written by an operation. There is no cgroup isolation, idle subtraction, bounded quiescence, or enforced live stop on wall/RSS/disk limits. Resource totals are post-run observations.

The workload count itself respects the narrow interpretation of the A0 budget: per layout there are 1,000 bootstrap inserts plus exactly 100 registered mutations (`29 INSERT / 30 UPDATE / 20 MERGE / 20 FORGET / 1 UPGRADE`). The run completed in 64.4 s with about 14.7 MiB output and 34.8 MiB RSS. However it ran under `/home/ubuntu/pz` on the system ext4 logical volume, not on a dedicated experiment NVMe as registered.

### 2.4 What the layout observations actually mean

`UPDATE < INSERT` and the A/B/C extent differences are facts of this specific extent proxy. They are strongly explained by SQLite B-tree/table touches, page allocation, one-transaction WAL framing, and omitted main-page overwrites. Layout C additionally retains immutable evidence versions while A/B overwrite current state, so all three variants do not have identical physical/history semantics.

The result can motivate a properly measured vertical-partitioning diagnostic, but the observed tradeoff is generic row-versus-field separation and does not yet support an agent-memory paper mechanism.

### 3.2 Capacity curves are hard-coded by admission behavior

Each new task creates 50 memories in fixed order; the first 10 are always “critical.” `init_task_memories` adds a new object to the active cache only when a free slot exists. When the cache is full, it does not admit the new item and evict an old item. `evict_to_capacity` only handles over-capacity state and therefore supplies no normal replacement policy.

Consequences:

* at capacity 50, the first task fills the cache and succeeds; the remaining four tasks cannot admit their memories, giving exactly `1/5 = 0.2`;
* after downward eviction, the cache remains full at each smaller capacity, so all new tasks fail;
* on the upward path, each capacity increase opens slots, and the first new task inserts its critical memories first, giving exactly `1/5 = 0.2` again.

The zero-variance success pattern across seeds is a structural artifact, not replication of a phenomenon.

### 3.3 The endogenous feedback loop is absent

The spec states that the query depends on previous action results and that action-created memories alter future requests. The implementation instead pre-generates every needed ID from `(task_id, seed)`; it never depends on the previous action. Derived memories are not referenced by any later query. Hence the required path

```text
miss -> action -> write -> future request distribution
```

does not exist in the executable model.

Transparent miss returns backing-store content by definition, while semantic miss returns `None` by definition. Their 100% success gap proves only that returning content differs from withholding it. It does not establish a closed-loop effect.

### 3.4 Fork and hysteresis comparisons are not paired

Fork A and B do not receive the same exogenous task sequence. A uses intermediate task IDs 15–24 and evaluation ID 125 repeatedly; after a shared counter is mutated, B uses intermediate IDs 25–34 and evaluation ID 135 repeatedly. A returns to capacity 30 with 20 empty slots, while B remains full. A's first evaluation task therefore admits its first ten critical memories and mechanically obtains the reported 0.2 advantage.

The required reset/crossing control is absent, and the raw memory-set sizes differ. Downward and upward curves also use different task IDs and accumulated states. The analysis says the upward path carries a “pruned memory set,” but the code never removes objects from `memory_set`; it only evicts from `active_cache`. Consequently neither the fork nor down/up comparison identifies hysteresis or creative destruction.
